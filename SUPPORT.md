# Getting Support

Thanks for using **opnDossier**. This page points you to the right place for whatever you need.

## Documentation first

Most questions are answered in the docs:

- **Documentation site:** <https://opndossier.com/>
- **CLI reference, output schemas, and exit codes:** [docs/for-agents.md](docs/for-agents.md)
- **Known gotchas and non-obvious behavior:** [GOTCHAS.md](GOTCHAS.md)

## Questions and discussion

For usage questions, configuration help, or to float an idea before filing an issue, use **[GitHub Discussions](https://github.com/EvilBit-Labs/opnDossier/discussions)**. Discussions are the best fit for open-ended conversations that aren't a confirmed bug or a concrete feature request.

## Reporting a bug or requesting a feature

Open a **[GitHub Issue](https://github.com/EvilBit-Labs/opnDossier/issues/new/choose)** and pick the matching template (bug, feature, or documentation). Before you do:

- Search [existing issues](https://github.com/EvilBit-Labs/opnDossier/issues) to avoid duplicates.
- Run the latest release where possible.
- **Sanitize any `config.xml` before attaching it** — use `opnDossier sanitize` so you never paste unredacted secrets into a public issue.

## Reporting a security vulnerability

**Do not open a public issue for security problems.** Follow the [Security Policy](SECURITY.md): use [GitHub Private Vulnerability Reporting](https://github.com/EvilBit-Labs/opnDossier/security/advisories/new) (preferred) or the encrypted email channel documented there.
